Triton 语义
================

Triton 在大多数情况下遵循 NumPy 的语义，仅有少量例外。本文档将介绍 Triton 支持的部分数组计算特性，并说明 Triton 语义与 NumPy 存在差异的地方。

类型提升
--------------

**类型提升**（Type Promotion）发生在对不同数据类型的张量进行操作时。对于与 `dunder 方法 <https://docs.python.org/3/reference/datamodel.html#emulating-numeric-types>`_ 相关的二元操作，以及三元函数 ``tl.where`` 的最后两个参数，Triton 会按照类型层次（dtype 的集合）自动将输入张量转换为共同的数据类型：``{bool} < {整型 dtype} < {浮点型 dtype}``。

算法规则如下：

1. **类别（Kind）** 若某个张量的 dtype 属于更高类别，则另一个张量将提升至该 dtype：``(int32, bfloat16) -> bfloat16``

2. **位宽（Width）** 若两个张量的 dtype 属于同一类别，但其中一个位宽更高，则另一个张量提升至该 dtype：``(float32, float16) -> float32``

3. **优先 float16** 若两个张量位宽相同、符号性相同但 dtype 不同（如 ``float16`` 和 ``bfloat16``，或不同的 ``fp8`` 类型），则两者均提升为 ``float16``：``(float16, bfloat16) -> float16``

4. **优先无符号** 否则（位宽相同，符号性不同），提升为无符号 dtype：``(int32, uint32) -> uint32``

当操作涉及标量时，规则略有不同。此处的标量指数值字面量、标记为 `tl.constexpr` 的变量，或两者的组合。这些标量以 NumPy 标量表示，类型为 ``bool``、``int`` 或 ``float``。

当操作涉及张量与标量时：

1. 若标量的类别低于或等于张量，则标量不参与提升：``(uint8, int) -> uint8``

2. 若标量的类别更高，则在 ``int32`` < ``uint32`` < ``int64`` < ``uint64``（整型）或 ``float32`` < ``float64``（浮点型）中选择能容纳该标量的最小 dtype，张量和标量均提升至该 dtype：``(int16, 4.0) -> float32``


广播
------------

**广播**（Broadcasting）允许对形状不同的张量进行操作，通过自动扩展其形状至兼容的尺寸，而无需复制数据。规则如下：

1. 若某个张量的形状维度更少，则在左侧补 1，直到两个张量的维度数相同：``((3, 4), (5, 3, 4)) -> ((1, 3, 4), (5, 3, 4))``

2. 两个维度兼容的条件为：两者相等，或其中一个为 1。大小为 1 的维度将扩展以匹配另一个张量的对应维度：``((1, 3, 4), (5, 3, 4)) -> ((5, 3, 4), (5, 3, 4))``


与 NumPy 的差异
----------------------

**整数除法采用 C 语义的舍入方式** 出于效率考虑，Triton 的运算符遵循 C 语义而非 Python 语义。因此，对于符号不同的整数，``int // int`` 实现的是 `向零舍入（C 语言风格） <https://en.wikipedia.org/wiki/Modulo#In_programming_languages>`_，而非 Python 中的向负无穷舍入。同样地，取模运算符 ``int % int``（定义为 ``a % b = a - b * (a // b)``）也遵循 C 语义而非 Python 语义。

值得注意的是，当所有输入均为标量时，整数除法和取模仍遵循 Python 语义。
